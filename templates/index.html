ACME Travels - API Reference Guide

Explore the ACME Travels API for managing vacation packages, customers, and more.

Overview

This API allows you to manage travel-related data through RESTful endpoints. You can create, retrieve, update, and delete data related to vacation packages, hotels, bookings, and customers.

Packages Endpoints

GET /packages

Retrieve a list of all vacation packages.

Example Response:

{
    "id": 1,
    "name": "Beach Escape",
    "duration": "7 days",
    "price": 1999.99,
    "description": "A 7-day beach vacation package"
}

POST /packages

Create a new vacation package.

Parameters:

Parameter Type Description Required
name string The name of the package Yes
duration string The duration of the package Yes
price number The price of the package Yes
description string A description of the package Yes

Customers Endpoints

GET /customers

Retrieve a list of all customers.

Example Response:

{
    "id": 3,
    "name": "John Doe",
    "email": "johndoe@example.com",
    "phone": "+123456789"
}

POST /customers

Add a new customer to the system.

Parameters:

Parameter Type Description Required
name string The name of the customer Yes
email string The email of the customer Yes
phone string The phone number of the customer No

Error Codes

The following table explains the common error codes that may be returned by the API:

Error Code Description Possible Cause Resolution
400 Bad Request Missing or invalid parameters in the request body. Check the API documentation for the correct parameters and ensure they are passed in the request.
401 Unauthorized Authentication failed or not provided. Ensure that the correct authentication token is passed with the request.
403 Forbidden The request is authenticated, but the user does not have permission to access the resource. Ensure that the user has the necessary permissions to access the resource.
404 Not Found The requested resource could not be found. Check the URL and ensure the resource ID is correct.
500 Internal Server Error An error occurred on the server while processing the request. Try again later or contact support if the issue persists.
